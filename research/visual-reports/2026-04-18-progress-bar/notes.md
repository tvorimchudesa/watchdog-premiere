# Progress bar поведение при импорте

**Дата:** 2026-04-18
**Связанные секции чеклиста:** §22 (initially), §14.6 (resulting fix)
**Контекст:** Юзер говорит что progress bar "мерцает" / "дёргается" при импорте.
Claude утверждает: `transition: width 0.3s` уже сглаживает, остаются только паузы
~1-2s между chunks. Нужно визуально подтвердить.

## Наблюдения (user)

- При Sync All на папке 157 файлов "что-то мерцает"
- После разметки скриншота: наш SheepDog bar справа — **ровный, не мерцает**
- Мелкое окно по центру ("Import Files...") всплывает и исчезает → **это и есть мерцание**
- Юзер предположил: "будто вызывается пер батч? Или пер файл?"

## Frames

- `frames/001.png` — момент до начала импорта, бара ещё нет
- `frames/002-009.png` — импорт идёт, наш bar стабильно заполнен ~45% ("70/157"),
  центральная модалка Premiere "Import Files..." периодически видна
- Аннотированный скриншот (полное разрешение, в сообщении чата):
  - Стрелка на SheepDog panel: "Наш бар, все с ним ок"
  - Стрелка на Premiere-модалку: "Видишь он будто бы вызывается пер батч? Или пер файл? И мерцает"

## Анализ (Claude)

**Root cause найден:**

Мерцание — **не наш bar**. Это **нативная модалка Premiere** "Import Files..."
(центр экрана, маленькое окно с собственным progress bar и Cancel). Premiere
показывает её автоматически на каждый вызов `app.project.importFiles()`.

Механика:
- У нас `DEFAULT_CHUNK_SIZE = 10` в [importer.js:20](sheepdog/js/modules/importer.js#L20)
- 157 файлов → 16 batch-ей → **16 раз** Premiere открывает и закрывает свою
  модалку (каждые ~1-2s)
- Юзер видит этот стробоскоп и воспринимает как "наш plugin мерцает"

**Подтверждение гипотезы пользователя:**
Да, вызывается **per batch** (не per file). Ответ: 16 раз на 157 файлов при
chunk=10.

**Наш SheepDog bar работает ровно** — юзер это подтвердил на скриншоте.
Паузы 1-2s между chunk-updates есть, но они проходят под `transition: width 0.3s ease`
и визуально читаются как плавное наполнение, не как рывки.

## Вывод

**Опровергнуто:** что мерцает наш bar.
**Подтверждено:** мерцает нативная модалка Premiere, 1 раз на chunk.

**Fix:** `app.project.importFiles(paths, suppressUI=true, bin, asStills)` в
[host.jsx:77-80](sheepdog/jsx/host.jsx#L77-L80) — второй параметр с `false` → `true`.
Per-call флаг, скоуп только наш. Ручные импорты юзера (File → Import) не трогает.

**Записано как §14.6** в MVP-CHECKLIST.md с тест-чеклистом.

**Следствие для §22:** задачи progress bar UX остаются валидными, но теперь это
чистые polish-улучшения, а не fix-и. Снимаем "мерцание" из контекста §22.

**Дополнительный side effect:** в [app.js](sheepdog/js/app.js) stall-текст
раньше писал `"this batch is slow (chunk X/Y)"` — leak-ал implementation detail.
Заменено на нейтральное `"Importing — still working..."` (в контексте §22 принципа).
