# Visual reports

Пространство для визуальных отчётов: кадры, скриншоты, короткие серии
картинок из видео. Используем когда текстом описать UX/поведение сложно —
"посмотри и скажи что видишь".

## Структура

```
visual-reports/
  README.md                          # этот файл
  YYYY-MM-DD-<topic-slug>/
    frames/                          # .png / .jpg — последовательность или скриншоты
      001.png
      002.png
      ...
    notes.md                         # контекст + наблюдения + анализ + вывод
```

## Workflow

1. Пользователь создаёт папку `YYYY-MM-DD-<topic>/`
2. Кладёт кадры в `frames/` — именование `001.png`, `002.png`, ... (для порядка)
3. Заполняет **Контекст** и **Наблюдения (user)** в `notes.md`
4. Claude читает, смотрит frames, дописывает **Анализ (Claude)** и **Вывод** в тот же `notes.md`
5. Итог: в relevant секцию MVP-CHECKLIST.md попадает отметка со ссылкой на отчёт

## Шаблон `notes.md`

```md
# <Короткая тема>

**Дата:** YYYY-MM-DD
**Связанные секции чеклиста:** §X.Y (ссылка)
**Контекст:** что тестировалось, какой сценарий воспроизводился, какая гипотеза

## Наблюдения (user)

- Что увидел / что ожидал / что смущает

## Frames

- `frames/001.png` — начало импорта
- `frames/010.png` — момент X
- ...

## Анализ (Claude)

- Что вижу в кадрах
- Как это соотносится с кодом / ожиданиями
- Подтверждает/опровергает гипотезу

## Вывод

- Подтверждено / опровергнуто: ...
- Root cause (если найден): ...
- Дальнейшие действия: ...
```

## Видео → кадры

Два лаунчера в `tools/` — выбирай по вкусу:

### Windows (drag-and-drop) — `video-to-frames.bat`

1. Перетащи видеофайл прямо на `tools/video-to-frames.bat` в Explorer.
2. Откроется окно: `How many frames? [default 20]:` — введи число или жми Enter.
3. Скрипт:
   - посчитает total frames через `ffprobe`
   - вычислит stride = total / count
   - положит кадры рядом с видео в `./frames/`
   - откроет папку с результатами

Альтернативно из cmd:
```cmd
tools\video-to-frames.bat video.mp4 20
```

### Git Bash / Mac / Linux — `video-to-frames.sh`

```bash
./tools/video-to-frames.sh video.mp4          # 20 кадров (default)
./tools/video-to-frames.sh video.mp4 40       # 40 кадров — подробнее
```

### Под капотом

Оба скрипта одинаковы: `ffprobe` → total frames → stride = total/count →
`ffmpeg -vf select='not(mod(n,stride))'`. Равномерная выборка по всей длине
видео, а не просто первые N кадров.

**Требует:** `ffmpeg` + `ffprobe` в PATH.
**Установка:** `winget install ffmpeg` / `choco install ffmpeg` / `scoop install ffmpeg`.

## Практические заметки

- **Большие серии (>50 кадров)** — если репо разрастается, рассмотрим git-lfs. Пока не требуется.
- **Приватные/чувствительные UI** — не пушим в public, перед commit проверяем что в кадрах нет имён клиентов / путей с персональными данными.
- **Короткие отчёты** — если достаточно 1-2 скриншотов без кадровой раскладки, `frames/` всё равно удобно держать (единообразие).
