# Screenshots

Static PNG mockups + design references. Snapshots from Figma, design tools,
and panel iterations — used to compare against rendered output of
`mockups/panel/panel.figma-script.js`.

## Notable

- **`mockup.png`** / **`mockup-current.png`** — primary panel mockup target +
  current state. Used as visual diff during reorgs (col-architecture commit).
- **`frame.png`** / **`frame-1.png`** / **`frame-2.png`** / **`frame-iter-2.png`** /
  **`frame-iter-3.png`** — Figma frames at various iteration points. `frame.png`
  is the original; `-1`/`-2` are reference snapshots; `-iter-2`/`-iter-3` are
  later variations.
- **`buttons.png`**, **`safety-cover.png`**, **`eye-table.png`** — component
  detail shots.
- **`advanced-mode.png`**, **`easy-mode.png`**, **`easy-mode-2.png`** — Simplified
  vs Advanced mode visual exploration (early — predates the v2 §2 mockup).
- **`file-manager.png`** / **`file-manager-current.png`** — Watchtower competitor
  reference (filesystem-side view).
- **`eye-disabled.png`** — small specimen of disabled eye state.

## Convention

Lower-kebab-case throughout. If a name needs disambiguation, use
`-iter-N` (iteration), `-current`, or `-N` suffix — never camelCase or spaces.
